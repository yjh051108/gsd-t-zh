#### The three guess-types (ANY triggers research/verification)
1. **Unknown** ŌĆö the agent lacks the fact.
2. **Assumed** ŌĆö asserting a shape/value never verified ("the create call returns a `url`" because it'd make sense, not because it was confirmed). Plausible ŌēĀ confirmed.
3. **Stale** ŌĆö knew it, but it's an external/time-varying fact with age (an API last seen months ago, a price, a model ID, a library signature). Was-true ŌēĀ is-true. **DEFAULT = FAIL TOWARD VERIFY:** any external/time-varying load-bearing fact without a FRESH cited source is treated as stale ŌåÆ search. Do NOT trust the agent's self-assessment of its own staleness ŌĆö that's another guess.

#### Goal
At every workflow phase (AND in direct conversation ŌĆö scope is both), for each load-bearing claim the agent tags KNOWN vs GUESSED (unknown/assumed/stale). A GUESSED claim is (a) CLASSIFIED internal-vs-external by a deterministic module, (b) external ŌåÆ a web-research `agent()` stage resolves it and CITES the fact + source URL into the artifact; internal ŌåÆ grep/Read; (c) the verify gate FAILS if a load-bearing external claim remains uncited (no silent guess). Result: confident-guess findings stop recurring because the fact is verified once and pinned.

#### The internal-vs-external classifier (deterministic, the load-bearing piece)
Given a GUESSED claim, it is EXTERNAL (ŌåÆ web) when it concerns a third-party API contract/endpoint/limit/return-shape/error-shape, a library/framework behavior or version, a platform/browser/runtime behavior, a standard/spec, or "current/latest." It is INTERNAL (ŌåÆ grep/Read/git, NEVER web) when it concerns this repo's own code/contracts/schema/ownership/sandbox/test architecture. The classifier routes by FEATURE CLASS (external-signal: third-party proper-noun OR an API/return-shape/limit/version/spec assertion about a system outside this repo ŌĆö so a proper-noun-LESS external guess like "max batch size the payments endpoint accepts" still routes external because it ASSERTS an external system's behavior unverified; internal-signal: repo path/symbol/"this repo/our/the existing"). Ambiguous ŌåÆ internal-first ŌåÆ grep ŌåÆ escalate to external only if grep returns nothing (owned by the wiring domains). Held-out generalization test guards against keyword-memorization.
